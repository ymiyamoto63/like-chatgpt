# サジェストAPI 契約文書

> 本書は `docs/requirements/input-autocomplete.md` に基づく、メッセージ入力補完用エンドポイント `POST /api/suggest` の契約文書である。

## 位置づけ

- 入力途中のテキストから後続の文を推測して返すAPI。現時点ではLLMには接続しておらず、バックエンドのモック実装（`MockSuggestService`）が定型文リストへの前方一致で決定論的に候補を返す。
- 将来、モック実装をLLM呼び出しに置き換える際も、リクエスト/レスポンスの形はこのまま維持し、フロントエンド（`frontend/src/api/suggestApi.ts`、`MessageInput.vue` のゴーストテキスト表示）には変更が不要となることを意図している。
- 補完対象の定型文は `frontend/src/constants/reportButtons.ts` のレポートボタン文言と一致させる（言語をまたぐため自動同期はできない。双方のソースにその旨のコメントあり）。

## リクエスト: `SuggestRequest`

| フィールド | 型 | 必須 | 説明・制約 |
|---|---|---|---|
| `text` | `string` | ○ | 入力途中のテキスト。空文字も許容する（候補なしとして200で返る。`/api/chat` と異なり400にはしない） |

```
POST /api/suggest
{
  "text": "当月担"
}
```

## レスポンス: `SuggestResponse`

| フィールド | 型 | 必須 | 説明・制約 |
|---|---|---|---|
| `completion` | `string \| null` | ○ | 補完候補の「残り文字列」。入力が定型文のいずれかに前方一致した場合、定義順で最初の1件の残り部分を返す。非一致・完全一致（残りが空）・空入力の場合は `null` |

```
200 OK
{
  "completion": "当者別問い合わせ"
}
```

候補なしの場合:

```
200 OK
{
  "completion": null
}
```

## フロントエンド側の扱い

- 入力停止から約300msのデバウンス後に呼び出す。IME変換中は呼び出さない。
- `completion` が非nullのとき、入力欄内にゴーストテキスト（薄色）として表示する。Tabで確定、Escで消去。
- 呼び出し失敗時は候補を表示しないだけで、エラー表示は行わない（補完は補助機能であり入力・送信を妨げない）。
