# 問い合わせ送信前のFAQ提示機能

## 概要

Issue #2「問い合わせ送信前のFAQ提示機能」の実装を完了しました。既存の新規問い合わせウィザード（カテゴリ→緊急度→内容入力→確認→完了）に、内容入力直後に FAQ 提示ステップを挿入します。選択済みカテゴリに対応するFAQタイトル3件をリンクとして表示し、クリックでFAQ詳細をAI回答として表示。FAQで解決した場合はフロー終了、解決しない場合は既存の確認ステップに合流して問い合わせ登録に進みます。

## 変更内容

### バックエンド
- **FAQリストコンポーネント実装**
  - 新規 `FaqListComponent.java` 作成: `type: "faq_list"`, `titles: string[]` を持つレコード
  - `UiComponent.java` に `@JsonSubTypes` で `faq_list` を追加

- **MockChatService フロー変更**
  - FAQデータ: 4カテゴリ（請求/技術/アカウント/その他）×3件をハードコード
  - 累積文（`カテゴリ: X / 緊急度: Y / 内容: Z`）への応答を確認ステップから **FAQ提示ステップ** に変更
  - 新メソッド: `faqPresentation()`, `faqDetailResponse()`, `faqResolvedCompletion()`
  - `inquiryFlowResponse()` 分岐順序をFAQ判定優先に変更

### フロントエンド
- **型と検証**
  - `types/chat.ts`: `FaqListComponent` インターフェース追加
  - `chatApi.ts`: `isValidFaqListComponent()` 関数追加

- **描画コンポーネント**
  - 新規 `FaqListView.vue`: FAQタイトルをリンク風テキスト（下線付き）で表示
  - `MessageBubble.vue`: 描画分岐を追加

- **ストア**
  - `conversationStore.ts`: `findLastInquirySummary()` で累積文を復元、「解決しないので問い合わせる」時に定型文を組み立て

### ドキュメント
- `docs/chat-response-schema.md`: `FaqListComponent` の契約定義追記

### テスト
- バックエンド: `MockChatServiceTest.java`, `ChatControllerTest.java` 更新
  - FAQ提示、FAQ詳細、4カテゴリ各3件、解決/未解決分岐の各テスト追加
